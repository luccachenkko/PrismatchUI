UPPGIFT: BYGG UI RUNT VÅRT BEFINTLIGA PRISMATCHNINGSPROJEKT

Projekt att utgå från:
Det här UI:t ska byggas ovanpå det befintliga projektet vi redan har tagit fram lokalt:

price-match-batch-mvp.zip

Projektet fungerar redan som en batch-MVP via terminal/CSV. Det kan:

1. Hämta Shopify access token via nya Shopify Dev Dashboard-flödet.
2. Hämta aktuellt Shopify-pris och eget Shopify-lager.
3. Läsa lokala produkter och konkurrentlänkar.
4. Hämta konkurrentpriser från verkliga produkt-URL:er.
5. Välja lägsta giltiga konkurrentpris per produkt.
6. Räkna föreslaget nytt pris.
7. Kontrollera marginalregel.
8. Skapa prisrapport.
9. Uppdatera Shopify först efter manuell godkänning.

UI:t ska alltså inte byggas som ett separat låtsasprojekt. Det ska hänvisa till, återanvända och bygga vidare på logiken i vårt befintliga projekt.

4. SHOPIFY-AUTENTISERING: NYA DEV DASHBOARD-FLÖDET
============================================================

Appen ska stödja nya Shopify Dev Dashboard-flödet.

Miljövariabler:

SHOPIFY_SHOP_DOMAIN=fristedt-handel-ab.myshopify.com
SHOPIFY_CLIENT_ID=...
SHOPIFY_CLIENT_SECRET=...
SHOPIFY_API_VERSION=2026-04
PRICE_FETCH_CONCURRENCY=5
SHOPIFY_UPDATE_CONCURRENCY=1

Viktigt:

- Client Secret får aldrig exponeras i frontend.
- Client Secret får aldrig skickas till browsern.
- Client Secret får aldrig commitas.
- .env ska ligga i .gitignore.

Backend ska hämta access token via:

POST https://{shop}.myshopify.com/admin/oauth/access_token

Body:

grant_type=client_credentials
client_id=SHOPIFY_CLIENT_ID
client_secret=SHOPIFY_CLIENT_SECRET

Svaret innehåller en access token.

Den token ska backend använda mot Shopify GraphQL Admin API:

https://{shop}.myshopify.com/admin/api/2026-04/graphql.json

Header:

X-Shopify-Access-Token: {access_token}

============================================================
9. PRISMATCHNINGSREGLER
============================================================

Reglerna ska köras i denna ordning.


REGEL 1: ENDAST AKTIVA PRODUKTER
--------------------------------

Om produkten eller regeln är avstängd ska den inte prismatchas.

Status:

SKIPPAD_INAKTIV


REGEL 2: KONTROLLERA EGET SHOPIFY-LAGER FÖRST
---------------------------------------------

Om eget Shopify-lager är 0 eller mindre ska produkten skippas.

Status:

SKIPPAD_EGET_LAGER_0

Orsak:

Produkten finns inte i lager i din Shopify-butik. Konkurrentlänkar hämtades inte.

Viktigt:

Om eget lager är 0 ska inga konkurrent-URL:er hämtas för den produkten.


REGEL 3: INKÖPSPRIS KRÄVS
-------------------------

Om produkten saknar inköpspris ska den inte prismatchas.

Status:

SAKNAR_INKOPSPRIS

Orsak:

Inköpspris saknas. Kan inte räkna marginal.


REGEL 4: MINST EN AKTIV KONKURRENTLÄNK KRÄVS
--------------------------------------------

Om produkten saknar aktiva konkurrentlänkar ska den inte prismatchas.

Status:

SAKNAR_LANKAR


REGEL 5: HÄMTA ENDAST PRIS FRÅN KONKURRENTER
--------------------------------------------

Scrapers ska endast hämta pris.

De ska INTE hämta, tolka eller använda konkurrentens lagerstatus.

Konkurrentens lagerstatus ska alltså INTE finnas med i:

- datamodell
- regler
- UI
- scraper-resultat
- rapporter


REGEL 6: VÄLJ BILLIGASTE GILTIGA KONKURRENTSPRIS
------------------------------------------------

För varje produkt:

1. Hämta pris från alla aktiva konkurrentlänkar.
2. Ignorera misslyckade hämtningar.
3. Ignorera pris null, 0 eller negativa priser.
4. Välj lägsta giltiga pris.

Om inget giltigt pris finns:

Status:

INGET_PRIS

Orsak:

Inget giltigt konkurrentpris kunde hämtas.


REGEL 7: RÄKNA FÖRESLAGET PRIS
------------------------------

Formel:

föreslaget_pris = billigaste_konkurrentpris - undercut_kr

Exempel:

Billigaste konkurrent: 1740 kr
undercut_kr: 1 kr
föreslaget pris: 1739 kr


REGEL 8: MINSTA MARGINAL
------------------------

Formel:

minsta_tillatna_pris = inkopspris / (1 - min_marginal_procent / 100)

Om:

föreslaget_pris < minsta_tillatna_pris

ska raden blockeras.

Status:

BLOCKERAD_MARGINAL

Orsak:

Föreslaget pris går under minsta tillåtna marginal.

Detta är den enda prisblockeringen som ska finnas.


REGEL 9: OM PRISET REDAN ÄR SAMMA
---------------------------------

Om föreslaget pris är samma som nuvarande Shopify-pris:

Status:

INGEN_ANDRING

Orsak:

Shopify-priset ligger redan på föreslaget pris.


REGEL 10: OK FÖR GODKÄNNANDE
----------------------------

En produkt får status OK om:

- produkten är aktiv
- produkten har eget Shopify-lager > 0
- inköpspris finns
- minst en aktiv konkurrentlänk finns
- minst ett giltigt konkurrentpris hittades
- föreslaget pris inte går under minsta marginal
- föreslaget pris skiljer sig från Shopify-priset

Status:

OK

============================================================
11. SHOPIFY-UPPDATERING
============================================================

Shopify ska inte uppdateras direkt efter prishämtning.

Rätt flöde:

1. Kör prismatchning.
2. Skapa rekommendationer.
3. Användaren granskar rapporten.
4. Användaren godkänner rader.
5. Systemet uppdaterar bara godkända OK-rader.

Shopify-uppdatering ska bara ske om:

recommendation.status = OK
approved = true

Om raden är blockerad, INGEN_ANDRING, SAKNAR_INKOPSPRIS, SKIPPAD_EGET_LAGER_0, INGET_PRIS eller annan icke-OK-status får den inte uppdateras.


============================================================
12. PRISUPPDATERING VIA SHOPIFY GRAPHQL
============================================================

Prisuppdatering ska ske via Shopify GraphQL Admin API.

Mutation:

productVariantsBulkUpdate

Den ska uppdatera variantens price.

Uppdateringen ska loggas i lokal databas:

- old_price
- new_price
- status
- error
- updated_at

- inStock
- availability
- stockStatus

Domänstyrning:

hemmabutiken.se -> hemmabutiken scraper
cdon.se -> cdon scraper
themobilestore.se -> themobilestore scraper

Om en URL har en domän utan scraper:

status: failed

2. Det finns ingen mockdata, fake-data, demodata eller hårdkodad produktdata.

============================================================
19. VIKTIG SLUTREGEL
============================================================

Detta ska byggas som ett riktigt internt affärsverktyg, inte som ett snyggt UI med låtsasdata.

Om Shopify inte fungerar ska UI visa fel.

Om databasen är tom ska UI visa tomt läge.

Om ingen produkt är synkad ska UI säga:

Inga produkter synkade ännu. Klicka på "Synka produkter från Shopify".

Inte visa exempelprodukter.

Om inga länkar finns för en produkt ska UI säga:

Inga konkurrentlänkar finns för denna produkt.

Inte visa exempel-URL:er.

Om en scraper saknas ska UI säga:

Ingen scraper finns för denna domän.

Inte gissa eller använda generic/fake-pris.

Detta är ett absolut krav.
